# Phase 08 Plan 01: Profiling Infrastructure

**Wave:** 1
**Depends on:** None (parallel-ready)
**Status:** Ready for execution
**autonomous:** true

---

## Goal

Add Rust timestamp logging to key init steps + frontend `performance.now()` profiling. Profile all three phases: Bun Sidecar startup, Rust initialization, React/Webview rendering.

---

## Context from Research

- Profiling infrastructure partially exists (`Instant::now()` in `sidecar.rs`, `performance.now()` in `useAutoScroll.ts`)
- Rust: Use `std::time::Instant::now()` + `ulog_info!("[startup:profile] ...")`
- Frontend: Use `performance.now()` + `console.log("[startup:profile] ...")`
- Only profile at major milestones (stage completions), not inside tight loops
- Frontend timing submitted to Rust via invoke for centralized logging (optional enhancement)

## Tasks

### Task 1: Rust Profiling in lib.rs

<read_first>
- `src-tauri/src/lib.rs` (lines 251-290, 369-461)
- `src-tauri/src/sidecar.rs` (Instant::now usage pattern)
</read_first>

<action>
1. Add `use std::time::Instant;` import at top of lib.rs
2. At start of `.setup()` closure, add:
   ```rust
   let startup_timer = Instant::now();
   ```
3. Before each `emit("startup:stage", ...)` call, record elapsed time:
   - Stage 1: After tray setup (line ~384)
   - Stage 2: Inside management_api spawn after OK (line ~441)
   - Stage 3: Inside cron spawn after initialize (line ~459)
4. At end of setup (before `Ok(())`), add summary log:
   ```rust
   ulog_info!("[startup:profile] setup phases done: plugin={}ms tray={}ms mgmt={}ms cron={}ms total={}ms",
       stage_timers.get("plugin").copied().unwrap_or(0),
       stage_timers.get("tray").copied().unwrap_or(0),
       stage_timers.get("management_api").copied().unwrap_or(0),
       stage_timers.get("cron").copied().unwrap_or(0),
       startup_timer.elapsed().as_millis());
   ```
5. Use `std::collections::HashMap<&str, u128>` or individual variables to track each stage timing

</action>

<acceptance_criteria>
- [ ] `grep -n "startup:profile" src-tauri/src/lib.rs` returns 4+ lines with timing data
- [ ] Each stage emit logs its elapsed time in milliseconds
- [ ] Final summary log shows all 4 stage times + total setup time
- [ ] Profiling does NOT add logging to tight loops or hot paths

### Task 2: Frontend Profiling in App.tsx

<read_first>
- `src/renderer/App.tsx` (startup:stage listener, useEffect mount)
- `src/renderer/hooks/useAutoScroll.ts` (performance.now usage pattern)
</read_first>

<action>
1. Add at top of App component or in a dedicated `useStartupProfiling` hook:
   ```typescript
   const startupStart = performance.now();
   ```
2. In each `startup:stage` event handler, log:
   ```typescript
   const elapsed = Math.round(performance.now() - startupStart);
   console.log(`[startup:profile] Stage ${stage} complete at ${elapsed}ms`);
   ```
3. When `startup:complete` fires (or stage 4), log:
   ```typescript
   const totalMs = Math.round(performance.now() - startupStart);
   console.log(`[startup:profile] Frontend startup: ${totalMs}ms`);
   ```
4. Optional: send to Rust via `invoke('cmd_log_startup_timing', { phase, elapsedMs })` for centralized log

</action>

<acceptance_criteria>
- [ ] `grep -n "startup:profile" src/renderer/App.tsx` returns 5+ lines
- [ ] Each stage emits a console.log with elapsed time
- [ ] Final startup complete emits total elapsed time
- [ ] No performance impact on hot paths (only stage milestones)
