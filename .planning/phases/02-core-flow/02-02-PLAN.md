<task type="auto">
  <name>Task 1: Create LoginPage.tsx</name>
  <files>src/renderer/pages/LoginPage.tsx</files>
  <action>
Create `src/renderer/pages/LoginPage.tsx` implementing login page per UI-SPEC.

**Interface:**
```typescript
interface LoginPageProps {
  isActive: boolean;
  onLoginSuccess: () => void;
}
```

**State:**
- `phone: string` - phone number input
- `code: string` - OTP code (from OtpInput)
- `countdown: number` - seconds remaining (0 = can send)
- `isSendingSms: boolean` - loading state for SMS
- `isSubmitting: boolean` - loading state for login
- `phoneError: string` - inline phone validation error
- `codeError: string` - inline OTP validation error

**Layout (per UI-SPEC Section 1.1):**
- Full-page overlay: `fixed inset-0 flex items-center justify-center z-[200] bg-black/30 backdrop-blur-sm`
- Card: `bg-[var(--paper-elevated)] border border-[var(--line)] rounded-[var(--radius-xl)] shadow-lg w-full max-w-md p-8`
- Title: `手机号登录` - `text-[var(--text-3xl)] font-semibold text-[var(--ink)] mb-6`
- Link to register: Top right - `text-[var(--ink-muted)] text-sm hover:text-[var(--ink)]`
- Footer link: `还没有账号？ 去注册` with `text-[var(--accent)]` link

**Form structure:**
```
[手机号输入框]                    - Label: 手机号, placeholder: 请输入手机号
[发送验证码] [验证码输入框(OTP)]   - Send button + OtpInput side by side
                                     Send button: ghost style, disabled during countdown
                                     OtpInput for 6-digit code
[        登录        ]            - Primary button, full width
```

**Phone input:**
- Label: `text-[var(--text-sm)] font-medium text-[var(--ink)] mb-2`
- Input: `bg-transparent border border-[var(--line)] rounded-[var(--radius-sm)] px-3 py-2.5 text-sm placeholder:text-[var(--ink-muted)] focus:border-[var(--accent)] focus:outline-none transition-colors`
- Validation: `^1[3-9]\d{9}$` - Error: `手机号格式不正确`

**Send SMS button:**
- Default: `bg-[var(--button-secondary-bg)] text-[var(--button-secondary-text)] rounded-full px-4 py-2.5 text-[13px] font-medium`
- Counting down: disabled, text: `重新发送 (${countdown}s)`, opacity-50
- Loading: show Loader2 spinner + `发送中...`

**SMS countdown logic:**
- After successful `sendSmsCode()`, start `useEffect` with `setInterval` at 1000ms
- `setCountdown(60)`, decrement each second
- Clear interval when countdown reaches 0 or component unmounts
- Only start countdown AFTER successful API response

**Login button:**
- `bg-[var(--button-primary-bg)] text-[var(--button-primary-text)] rounded-full px-6 py-2.5 text-[13px] font-medium hover:bg-[var(--button-primary-bg-hover)] disabled:opacity-50`
- Loading: show Loader2 spinner + `登录中...`
- Disabled if phone is empty or code.length < 6 or isSubmitting

**Error handling:**
- Phone validation on blur: if invalid, show inline error below input
- OTP errors from API: catch and show inline error below OtpInput
- Network errors: show toast via `useToast().error()`
- Error display: `text-[var(--text-sm)] text-[var(--error)] mt-1` with AlertCircle icon

**Error message mapping (from UI-SPEC):**
- 400 + "Invalid phone number": `手机号格式不正确`
- 400 + "Invalid verification code": `验证码错误，请重新输入`
- 401: `验证码已过期，请重新获取`
- 429: `发送次数过多，请稍后再试`
- Network error: `网络连接失败，请检查网络设置`
- 500: `服务器错误，请稍后重试`

**On submit:**
1. Validate phone format → show phoneError if invalid
2. Validate code.length === 6 → show codeError if invalid
3. Set `isSubmitting = true`
4. Call `login(phone, code)` from useAuth()
5. On success: show toast `登录成功`, call `onLoginSuccess()`
6. On error: show appropriate error message, set `isSubmitting = false`

**Redirect to register:**
- Footer link `去注册` calls `dispatchEvent(new CustomEvent('navigate-to-register'))`
- App.tsx listens for this and creates new tab with `view: 'register'`
</action>

  </verify>
