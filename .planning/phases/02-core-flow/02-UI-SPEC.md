| Role | Value | CSS Variable | Usage |
|------|-------|--------------|-------|
| Dominant (60%) | `#faf6ee` / `#fffcf7` | `--paper` / `--paper-elevated` | Page background, card backgrounds |

**Accent reserved for**:
- Primary CTA buttons (登录 / 注册 / 发送验证码)

- Active tab indicators
- Links (去注册 / 去登录)

**Text colors**:
- Primary text: `--ink` (#1c1612) — headings, body text
- Secondary text: `--ink-secondary` (#2e2825) — subtitles
- Muted text: `--ink-muted` (#6f6156) — descriptions, placeholders, timestamps
- Subtle text: `--ink-subtle` (#a69a90) — disabled states, hints

**Source**: `--ink-*`, `--paper-*`, `--accent-*`, `--error`, `--success`, `--warning` tokens in `src/renderer/index.css` lines 270-314.

**Semantic color usage for Phase 2**:
- **Error states** (`--error`): Invalid phone format, wrong verification code, expired code, network errors
- **Success states** (`--success`): Login/register success toast, code sent confirmation
- **Warning states** (`--warning`): SMS rate limit reached (60s countdown), resend unavailable

## Component Specifications

### 1. Login Page (`LoginPage.tsx`)

**Layout**: Centered card on Launcher-style background (orbs + grid pattern)

**Card structure**:
```
┌────────────────────────────────────┐
│  nova-agents        去注册 (link)  │
│  ─────────────────────────────────  │
│                                     │
│  手机号登录                          │
│                                     │
│  [手机号输入框]                      │
│  [发送验证码] [验证码输入框]         │
│                                     │
│  [        登录        ]             │
│                                     │
└────────────────────────────────────┘
```

**CSS classes**:
- Container: `fixed inset-0 flex items-center justify-center z-[200]` (overlay style, same as ConfirmDialog)
- Card: `w-full max-w-md glass-card p-8` (glass-card = `bg-[var(--paper-elevated)] border border-[var(--line)] rounded-[var(--radius-xl)] shadow-lg`)
- Title: `text-[var(--text-3xl)] font-semibold text-[var(--ink)] mb-6`
- Form labels: `text-[var(--text-sm)] font-medium text-[var(--ink)] mb-2`
- Input fields: Follow Settings panel pattern — `bg-transparent border border-[var(--line)] rounded-[var(--radius-sm)] px-3 py-2.5 text-sm placeholder:text-[var(--ink-muted)] focus:border-[var(--accent)] focus:outline-none transition-colors`
- Primary button: `bg-[var(--button-primary-bg)] text-[var(--button-primary-text)] rounded-full px-6 py-2.5 text-[13px] font-medium hover:bg-[var(--button-primary-bg-hover)] disabled:opacity-50 flex items-center justify-center gap-1.5`
- Send SMS button: Ghost style — `bg-[var(--button-secondary-bg)] text-[var(--button-secondary-text)] rounded-full px-4 py-2.5 text-[13px] font-medium hover:bg-[var(--button-secondary-bg-hover)] disabled:opacity-50`

**Spacing**:
- Card padding: `p-8` (32px)
- Form section spacing: `space-y-6` (24px between fields)
- Label to input: `mb-2` (8px)
- Button spacing: `mt-8` (32px before submit button)

### 3. OTP Input Component (`OtpInput.tsx`)

**Specification**:
- 6 individual input boxes (single digit each)
- Auto-focus next input on digit entry
- Support paste (6-digit string from clipboard)
- Backspace handling: focus previous input and delete
- Maximum width per box: `w-12` (48px)
- Height: `h-14` (56px)
- Gap between boxes: `gap-2` (8px)

**CSS classes**:
- Container: `flex gap-2`
- Input box: `w-12 h-14 text-center text-2xl font-semibold border border-[var(--line)] rounded-[var(--radius-sm)] focus:outline-none focus:ring-2 focus:ring-[var(--accent)] focus:border-[var(--accent)] bg-[var(--paper)] transition-all`
- Filled state: `bg-[var(--paper-inset)] border-[var(--line-strong)]`
- Error state: `border-[var(--error)] focus:ring-[var(--error)]`

**Interaction states**:
- Default: Border `var(--line)`, transparent background
- Focus: Accent ring (2px), accent border
- Filled: Slightly darker background
- Error: Red border and ring

### 4. Loading States

**Button loading** (following ConfirmDialog pattern):
- Show `<Loader2 className="h-3 w-3 animate-spin" />` icon
- Button text changes to "发送中..." / "登录中..." / "注册中..."
- Button disabled with `disabled:opacity-50`
- Cursor: `cursor-wait`

**Page loading** (initial auth check):
- Full-page spinner: `fixed inset-0 flex items-center justify-center bg-[var(--paper)] z-[300]`
- Spinner: `<Loader2 className="h-8 w-8 animate-spin text-[var(--accent)]" />`

### 5. Error Display

**Inline errors** (below input fields):
- Text: `text-[var(--text-sm)] text-[var(--error)] mt-1`
- Icon: `<AlertCircle className="h-3.5 w-3.5 inline-block mr-1" />`
- Example: "验证码错误，请重新输入"

**Toast errors** (via `useToast`):
- Network errors, rate limiting, server errors
- Auto-dismiss after 5 seconds
- Icon: `<AlertCircle className="h-4 w-4" />` (red)

### 6. SMS Countdown Timer

**Button states**:
- Default (countdown = 0): "发送验证码", enabled, `bg-[var(--button-secondary-bg)]`
- Counting down (countdown > 0): "重新发送 ({countdown}s)", disabled, `opacity-50 cursor-not-allowed`
- Loading: `<Loader2 className="h-3 w-3 animate-spin mr-1" />` + "发送中..."

**Timer implementation**:
- `useState` for countdown (0-60)
- `useEffect` with `setInterval` (1000ms)
- Clear interval on unmount
- Start countdown only after successful `sendSmsCode()` response

### 7. Logout Flow

**Trigger**: Settings page → "退出登录" button (or keyboard shortcut)

**UX**: Instant logout, no confirmation dialog (per RESEARCH.md line 547)
- Button: Ghost style with `text-[var(--error)] hover:bg-[var(--error-bg)]`
- Loading: Show Loader2 spinner, change text to "退出中..."
- Success: Toast "已退出登录", redirect to Launcher

**Multi-tab sync**:
- Dispatch custom event `window.dispatchEvent(new CustomEvent('auth:logout'))`
- All tabs listen for event and re-check auth state via `AuthContext`

## Interaction Patterns

### 1. Focus Management

**Page load**: Auto-focus phone input field
- Use `useRef` + `useEffect` to call `inputRef.current?.focus()`
- Prevent auto-focus if tab is not active (`isActive` prop from TabContext)

**OTP input**: Auto-focus next box
- On digit entry: `nextInput?.focus()`
- On backspace: `prevInput?.focus()`
- On paste: Focus last filled box

**Error handling**: Focus first error field
- After validation failure, scroll to and focus first invalid input
- Add `ring-2 ring-[var(--error)]` to indicate error

### 2. Keyboard Shortcuts

| Shortcut | Action | Scope |
|----------|--------|-------|
| Enter | Submit form (login/register) | Login/Register pages |
| Escape | Clear form / go back | Login/Register pages |
| Tab | Navigate between fields | All forms |
| Ctrl/Cmd+V | Paste OTP code | OTP input only |

### 3. Validation Feedback

**Phone number**:
- Format: 11 digits, starts with 1 (Chinese mobile)
- Validation: `^1[3-9]\d{9}$`
- Error: "手机号格式不正确" (inline, below input)

**OTP code**:
- Format: 6 digits
- Validation: `^\d{6}$`
- Error: "验证码错误，请重新输入" (inline, below OTP input)

**Username** (register only):
- Format: 2-20 characters, alphanumeric + Chinese + underscore
- Validation: `^[\u4e00-\u9fa5a-zA-Z0-9_]{2,20}$`
- Error: "用户名长度为2-20个字符" (inline, below username input)

**Real-time validation**:
- Validate on blur (field loses focus)
- Re-validate on change if error already shown
- Clear error when user starts typing

### 4. Loading State Management

**Per-operation loading** (local state):
- `isSendingSms` — Send SMS button only
- `isSubmitting` — Submit button only
- Both can be true simultaneously (user can click submit while SMS is sending)

**Global loading** (AuthContext):
- `isLoading` — Initial auth check on app load
- Show full-page spinner if `isLoading === true`

### 5. Redirect Behavior

**After successful login**:
- Redirect to Launcher page (home)
- Show success toast: "登录成功"
- Close login tab

**After successful registration**:
- Redirect to Launcher page
- Show success toast: "注册成功"
- Close register tab

**After logout**:
- Redirect to Launcher page
- Show toast: "已退出登录"
- All tabs reflect logged-out state

## Responsive Design

**Mobile breakpoint**: 768px (Tailwind `md:`)

**Desktop layout** (≥768px):
- Login/Register card: Centered, `max-w-md` (448px)
- Full-screen overlay style (same as ConfirmDialog)

**Mobile layout** (<768px):
- Card: `w-full max-w-full mx-4` (full width with margins)
- Padding: `p-6` (24px) instead of `p-8` (32px)
- Input font size: 16px (prevent iOS zoom)

**Touch targets**:
- Minimum button height: 44px (WCAG 2.1 AAA)
- OTP input boxes: Increase to `h-16` (64px) on mobile
- Spacing: `gap-3` (12px) between OTP boxes on mobile

## Accessibility

**ARIA labels**:
- Phone input: `aria-label="手机号"`
- OTP input: `aria-label="验证码"`
- Username input: `aria-label="用户名"`
- Send SMS button: `aria-label="发送验证码"`
- Submit button: `aria-label={type === 'login' ? '登录' : '注册'}`

**Keyboard navigation**:
- All inputs reachable via Tab key
- Focus visible: `focus-visible:ring-2 focus-visible:ring-[var(--accent)]`
- Skip links: Not needed (simple single-page forms)

**Screen reader support**:
- Error messages: `role="alert" aria-live="assertive"`
- Loading states: `aria-busy="true"` on forms during submission
- Countdown timer: `aria-label="重新发送验证码，还需等待 {countdown} 秒"`

**Color contrast**:
- All text meets WCAG 2.1 AA (4.5:1 for body text)
- Error messages: `--error` (#dc2626) on white background = 7.1:1 (AAA)
- Success messages: `--success` (#2d8a5e) on white background = 4.6:1 (AA)
