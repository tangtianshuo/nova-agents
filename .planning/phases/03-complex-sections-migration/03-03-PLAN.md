## Inline Delete Confirmation (Phase 3 requirement)

Since Phase 4 handles full dialogs (DeleteConfirmDialog), Phase 3 implements a simple inline delete confirmation:

- When user clicks delete button, show inline confirmation row below actions
- Confirmation row shows: "确认删除?" text + "取消" button + "删除" danger button
- Clicking "取消" hides the confirmation
- Clicking "删除" calls onDelete(provider) and hides confirmation
- Does NOT use a modal/dialog overlay — just inline expand below the card actions
