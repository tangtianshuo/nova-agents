## Typography

**Maximum 4 font sizes, 2 font weights** (per design checker constraint):

| Role | Size | Weight | Line Height | CSS Variable |
|------|------|--------|-------------|--------------|
| Page title | 28px | 600 (semibold) | 1.2 | `--text-3xl` |
| Section heading | 18px | 600 (semibold) | 1.4 | `--text-lg` |
| Body | 16px | 400 (normal) | 1.6 | `--text-base` |
| Button/Label | 13px | 500 (medium) | 1.5 | `--text-sm` |

**Usage in Phase 3**:
- Card title: `--text-sm` (13px, semibold) — provider/server name
- Card description: `--text-xs` (12px) with `--ink-muted` — model lists, descriptions
- Badge text: `--text-xs` (11px) — protocol tags, status badges

**Weight hierarchy** (2 weights + button exception):
- Weight 600 (semibold): Page title, section heading, card titles, emphasized labels
- Weight 400 (normal): Body text, descriptions
- Weight 500 (medium): Button text — exception per design system spec (design_guide.md Button规范 specifies font-medium)

## Visual Focal Point

**Primary visual anchor on Provider/MCP sections**: The section title (e.g., "模型供应商") at the top-left establishes context. The "添加" CTA button at top-right draws attention for the primary action. Cards use subtle hover states to indicate interactivity without competing with the CTA.

## Interaction Patterns

### 1. Card Hover States

**ProviderCard**:
- Hover: Border color changes from `--line` to `--line-strong`
- No translate or shadow changes (keeping it subtle)

**McpServerCard**:
- Hover: Border color changes from `--line` to `--line-strong`
- Settings button becomes more visible on hover (opacity change)

### 2. Toggle Interaction

**MCP Server Toggle**:
1. User clicks toggle
2. If disabling: Immediately toggle off, show toast "MCP 已禁用"
3. If enabling:
   - Show loading spinner in toggle
   - If needs config (API key missing): Show toast "请先配置 {name}（点击设置）", auto-open settings dialog
   - If successful: Toggle on, show toast "MCP 已启用"
   - If failed: Toggle stays off, show error toast

### 3. API Key Verification

**ProviderCard API Key**:
1. User types in API key input
2. On blur or Enter: Auto-save (debounced)
3. Verification status shown via:
   - Idle: No indicator
   - Loading: Spinner + "验证中..."
   - Valid: Green check + "已验证"
   - Invalid: Red X + error message below input

### 4. Delete Flow

1. User clicks delete button on ProviderCard
2. ConfirmDialog appears with danger styling
3. User confirms: Provider deleted, show toast "供应商已删除"
4. User cancels: Dialog closes, no action

## Responsive Design

**Breakpoint**: 768px (mobile/tablet boundary)

**Desktop layout** (>=768px):
- Provider/MCP grid: `grid-cols-2` (two columns)
- Container max-width: `max-w-4xl` (896px)
- Horizontal padding: `px-8` (32px)

**Mobile layout** (<768px):
- Provider/MCP grid: `grid-cols-1` (single column)
- Container max-width: `max-w-full` with `mx-4` margins
- Horizontal padding: `px-4` (16px)

**Touch targets**:
- Minimum button size: 44px height (WCAG 2.1 AAA)
- Toggle switch: 44px x 44px touch target
- Settings/delete buttons: Adequate padding for touch

## Accessibility

**ARIA labels**:
- Add button: `aria-label="添加供应商"` / `aria-label="添加 MCP 服务器"`
- Settings button: `aria-label="管理"` / `aria-label="设置"`
- Delete button: `aria-label="删除"`
- Toggle: `aria-label={isEnabled ? "已启用" : "已禁用"}`
- API Key input: `aria-label="API 密钥"`

**Keyboard navigation**:
- All buttons reachable via Tab
- Toggle: Space/Enter to toggle
- Focus visible: `focus-visible:ring-2 focus-visible:ring-[var(--accent)]`

**Screen reader support**:
- Provider cards: Announce provider name, type, verification status
- MCP cards: Announce server name, enabled/disabled state, config needed warning
