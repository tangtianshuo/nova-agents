### Reporting a Vulnerability

If you discover a security vulnerability, please report it by emailing **nova-agents.io@gmail.com**.

**Please do NOT:**
- Open a public GitHub issue for security vulnerabilities
- Disclose the vulnerability publicly before it's fixed

**Please include:**
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes (optional)

### Response Timeline

- **Initial response:** Within 48 hours
- **Status update:** Within 7 days
- **Fix timeline:** Depends on severity

### Security Best Practices

When using nova-agents:

1. **API Keys**: Store API keys securely. nova-agents stores them locally in `~/.nova-agents/`
2. **Project Access**: Only grant access to trusted projects
3. **Tool Permissions**: Review tool permission requests carefully
4. **Updates**: Keep nova-agents updated to the latest version

### 支持的版本

| 版本 | 支持状态 |
| ------- | ------------------ |

### 报告漏洞

如果您发现安全漏洞，请发送邮件至 **nova-agents.io@gmail.com**。

**请勿：**
- 为安全漏洞创建公开的 GitHub issue
- 在修复前公开披露漏洞

**请包含：**
- 漏洞描述
- 复现步骤
- 潜在影响
- 建议的修复方案（可选）

### 响应时间

- **初次响应：** 48 小时内
- **状态更新：** 7 天内
- **修复时间：** 取决于严重程度

### 安全最佳实践

使用 nova-agents 时：

1. **API 密钥**：安全存储 API 密钥。nova-agents 将其本地存储在 `~/.nova-agents/`
2. **项目访问**：仅授权访问可信项目
3. **工具权限**：仔细审查工具权限请求
4. **更新**：保持 nova-agents 更新到最新版本
